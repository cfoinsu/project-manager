# Project Atlas (forder) 개발 및 환경 구성 요약서

본 문서는 다른 PC에서 프로젝트 작업을 온전히 이어나갈 수 있도록 기술 스택, 디렉토리 아키텍처, 데이터 저장 방식 및 실행 명령어, 그리고 최근 변경 내역을 총망라하여 정리한 구성 요약서입니다.

---

## 1. 프로젝트 개요 (Overview)
Project Atlas는 로컬 폴더의 물리적인 디렉토리 구조를 분석하고, 정의된 프로세스 단계(Gantt/칸반)와 비교하여 필수 문서 파일의 누락 여부를 판별함과 동시에, 전사 인력의 프로젝트 투입공수(Workload) 및 배정 현황(Assignment)을 통합 관리하는 **Tauri 기반의 데스크톱 ERP 대시보드 애플리케이션**입니다.

---

## 2. 기술 스택 (Technology Stack)

### Front-End (Client)
*   **Core**: React v19, TypeScript
*   **Build Tool**: Vite v8 (기동 포트: `5174` 강제 지정)
*   **State Management**: Zustand v5
*   **Styling**: TailwindCSS v4, Lucide React (아이콘)
*   **Visualization**: React Flow (프로세스 플로우 맵), Recharts (차트 및 투입량 분석)

### Back-End (API Server)
*   **Runtime**: Node.js (Express v4)
*   **Database**: SQLite3 (`server/local.db`)
*   **Auth**: JWT (JSON Web Token), bcryptjs 비밀번호 단방향 해시 암호화
*   **Middleware**: CORS, Multer (업로드)

### Desktop Wrapper (Tauri Shell)
*   **Framework**: Tauri v1.6 (Rust 기반 로컬 파일 시스템 제어 및 로컬 SQLite 연동)
*   **Database**: SQLite (`src-tauri/src/db.rs` 및 `db_` 계열 명령어 연동)

---

## 3. 데이터 아키텍처 및 연동 모드 (Unified Data Layer)
`src/utils/api.ts`를 통해 프런트엔드는 서버 유무 및 기동 환경에 따라 아래 3가지 모드로 유연하게 동작합니다.

1.  **서버 모드 (Server Mode - Express)**
    *   Express 서버(`http://localhost:5000`)에 REST API로 통신하여 전사 데이터를 통합 공유 및 관리합니다. JWT 기반 인증 토큰이 `localStorage`에 관리됩니다.
2.  **Tauri SQLite 모드 (Offline Mode - Tauri)**
    *   Tauri 데스크톱 앱 내에서 동작할 때, Express 서버 오프라인 시 Rust 단의 SQLite 로컬 DB로 자동 폴백하여 데이터를 로컬 파일 시스템에 적재합니다.
3.  **로컬 웹 시뮬레이션 모드 (Local Fallback - Browser)**
    *   서버도 없고 Tauri가 아닌 일반 웹 브라우저 단독 기동 시, `localStorage` 기반의 데이터 시뮬레이션을 지원하여 완벽한 UI 구동을 보장합니다.

## 4. 디렉토리 구조 (Directory Structure)

```bash
forder/
├── package.json               # 프런트엔드 의존성 및 스크립트 정의
├── tailwind.config.js         # TailwindCSS 테마 설정 (Pretendard 등 폰트, 토스/노션 색상 정의)
├── vite.config.ts             # Vite 설정 (Vite 서버 포트 5174 고정)
├── server/                    # Express 백엔드 API 서버 폴더
│   ├── package.json           # 백엔드 의존성 및 스크립트
│   ├── index.js               # 백엔드 진입점
│   ├── db.js                  # 백엔드 SQLite 테이블 생성 및 쿼리 연동
│   └── local.db               # 백엔드 SQLite 파일 데이터베이스
├── src-tauri/                 # Tauri 데스크톱 래퍼 Rust 소스 폴더
│   ├── src/db.rs              # Rust 단 SQLite 접속, 테이블 마이그레이션 및 Tauri Command 바인딩
│   └── src/main.rs            # Tauri 앱 핸들러 및 앱 진입점
└── src/                       # React 프런트엔드 소스 폴더
    ├── types.ts               # Project, Process, Task, Assignment, User 등 인터페이스 정의
    ├── index.css              # 프리미엄 UI 스타일(모서리 24px 카드, 글래스모피즘 사이드바 등) 정의
    ├── utils/
    │   ├── api.ts             # REST API 요청, 로컬 폴백 연동, 통합 배정/인원 조회 헬퍼
    │   ├── db.ts              # 로컬 Tauri SQLite 연동 브릿지 유틸리티
    │   └── tauriBridge.ts     # Tauri API 및 로컬 탐색기(openInExplorer) 호출 유틸리티
    ├── store/
    │   ├── authStore.ts       # 로그인, 회원가입, 세션 체크, 로컬 폴백 상태 관리 (Zustand)
    │   └── projectStore.ts    # 활성 프로젝트 정보, 파일 분석, 진단, 칸반 작업 변경 등 관리 (Zustand)
    └── components/            # 주요 뷰 컴포넌트 폴더
        ├── ProjectOverview.tsx        # [최근 리디자인] 종합 개요 대시보드 (투입인원 현황판 포함)
        ├── DashboardView.tsx          # 메인 대시보드 프로젝트 목록 및 일정 Gantt 차트
        ├── AssignmentManagementView.  # 배정현황 관리 (투입 가능/불가 테이블 분리, 상태 필터링)
        ├── TaskBoardView.tsx          # 작업 보드 칸반 및 시간 지정 팝업
        └── DocumentLibraryView.tsx    # 필수 산출물 현황 관리
```

## 5. 최근 작업 및 주요 패치 사항 (Recent Updates)

*   **프로젝트 설명(`description`) 필드 탑재**:
    *   SQLite `projects` 테이블에 `description TEXT` 컬럼 추가 마이그레이션.
    *   Tauri Rust Struct 바인딩, Express API, Zustand 스토어 및 UI 갱신을 통해 실시간 설명 수정 및 로컬 영구 저장을 완벽히 구현했습니다.
*   **종합개요 UI 전면 개선 및 투입 인력 현황판 신설**:
    *   KPI 카드 요약 스펙을 프리미엄 룩앤필로 리디자인하고, 2열 카드 그리드를 3열로 확장하여 우측에 **'프로젝트 투입 인력 현황판'** 카드를 신설했습니다.
    *   사용자명 해시 함수를 이용한 자동 파스텔 그라디언트 이니셜 아바타와 직책, 역할, 투입률 배지(100% 초과 시 로즈색, 80% 이상 블루색 등 구분)를 연동했습니다.
    *   사용자 피드백을 수용하여 종합개요 헤더는 Carbon/Atlas 타이틀 및 D-Day 배지 구조로 복원하고, 박스 테두리를 걷어내어 부드러운 하단 슬레이트 경계선(`border-b`)으로 정보를 연결했습니다.
*   **화면 스크롤링 및 포트 고정**:
    *   종합개요 뷰포트 영역의 높이가 초과될 경우 화면이 스크롤되도록 `cds--overview-container` 스크롤 콘텍스트를 복구했습니다.
    *   `vite.config.ts` 설정을 보완하여 로컬 기동 시 항상 `http://localhost:5174` 포트로 웹 서버가 열리도록 조치했습니다.
*   **배정현황 관리 분리 및 필터**:
    *   사용자별 누적 투입률에 따라 **'투입 불가 인원(누적 100% 이상)'**과 **'투입 가능 인원(100% 미만)'**으로 배정 테이블 목록을 분리하고 프로젝트 상태 필터 탭을 신설했습니다.

## 6. 다른 PC에서 개발 환경 세팅 및 실행법

### 사전 준비 (Prerequisites)
*   **Node.js LTS** 설치
*   **Rust 및 C++ 빌드 도구** 설치 (Tauri 데스크톱 빌드를 할 경우에만 필수)

### 1) 패키지 설치
프로젝트 루트 폴더 및 서버 폴더로 진입해 패키지를 설치합니다.
```bash
# 1. 프런트엔드 패키지 설치
npm install

# 2. Express 백엔드 패키지 설치
cd server
npm install
cd ..
```

### 2) 백엔드 API 서버 실행
백엔드 SQLite 데이터베이스 및 JWT 로그인 통신을 기동하기 위해 서버를 실행합니다.
```bash
cd server
npm run start
# 또는 노드몬 개발 모드 기동 시: npm run dev
```

### 3) 프런트엔드 웹 서버 실행 (브라우저 확인)
포트 `5174`로 실행되어 브라우저에서 개발 모드로 확인이 가능합니다.
```bash
npm run dev
```

### 4) Tauri 데스크톱 래퍼 실행 (데스크톱 앱 기동)
Tauri 클라이언트를 데스크톱 앱 창으로 실행합니다. (로컬 파일 진단 가능)
```bash
npx tauri dev
```
