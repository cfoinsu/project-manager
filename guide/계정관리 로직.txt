# Authentication & Device Registration System

## 개요

Project Atlas는 로컬 SQLite 기반 데스크톱 애플리케이션이다.

외부 서버, 이메일 인증, SMS 인증은 사용하지 않는다.

인증은 다음 3단계로 구성한다.

- ID
- Password
- Registered Device

로그인 성공 조건

- 아이디 일치
- 비밀번호 일치
- 등록된 PC(Device) 일치

---

## 사용자 권한

### Administrator (admin)

권한

- 사용자 생성
- 사용자 수정
- 사용자 비활성화
- 비밀번호 초기화
- 등록 PC 초기화
- 사용자 권한 관리

### User

권한

- 로그인
- 비밀번호 변경
- 프로젝트 사용

제한

- 비밀번호 초기화 불가
- 등록 PC 변경 불가

---

## Device Registration System

### Device ID 생성

Tauri 환경에서 현재 PC의 고유 식별값을 생성한다.

수집 가능한 정보 예시

- Machine GUID
- Computer Name
- OS Installation ID
- Hardware UUID

실제 저장 방식

DEVICE_HASH 생성

SHA256 해시 처리 후 저장

원본 식별값 저장 금지

예시

{
  userId: 1,
  deviceHash: "A91D4B8..."
}

현재 Device Hash 저장

사용자 계정과 연결

6. 로그인 성공

## 목표

Project Atlas는 로컬 환경에서도 보안을 확보하기 위해

계정
+
bcrypt 비밀번호
+
등록된 PC(Device Hash)
+
관리자 승인

구조를 사용한다.

사용자는 등록된 PC에서만 로그인 가능하며,
비밀번호 초기화 및 장치 초기화는 관리자만 수행할 수 있다.
