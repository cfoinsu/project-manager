Folder Atlas 사용 가이드

프로젝트 폴더를 분석해 진척·산출물·인력을 한 곳에서 관리하는 데스크톱 프로젝트 매니저입니다.
이 문서는 어떤 버튼을 어떻게 누르는지 화면 순서대로 설명합니다.

버전 0.2.2 · 설치 불필요(단독 실행) · 데이터는 내 PC에 저장

0 설치와 로그인

새 노트북에 아무것도 설치돼 있지 않아도 됩니다. 실행 파일 하나로 동작합니다.

실행하기

  1. 받은 Folder Atlas_0.2.2_Portable.exe 를 더블클릭합니다. 설치 불필요
  2. “Windows의 PC 보호” 파란 창이 뜨면 추가 정보 → 실행 을 누릅니다. (사내용 미서명 앱이라 정상입니다.)
ℹ️
정식 설치를 원하면 setup.exe 또는 .msi 파일을 더블클릭하세요. Windows 11은 별도 준비가 필요 없습니다.

로그인

  1. 아이디 (또는 이메일) 칸에 아이디를 입력합니다.
  2. 비밀번호 칸에 비밀번호를 입력합니다.
  3. 로그인하기 버튼을 클릭합니다.
  4. 처음 로그인하면 “기기 등록 및 보안 동의” 창이 뜹니다 → 동의 및 PC 등록 클릭. (계정당 1대 PC만 등록됩니다.)

1.6 역할별 작업 · 확인 · 효용가치(ROI) ★

이 도구의 목적은 "정보를 찾고 취합하는 시간"을 줄여 그 시간을 실제 생산 업무로 돌리는 것입니다. 역할마다 무엇을 하고(작업) → 무엇을 보고 판단하며(확인) → 그것이 어떤 낭비를 줄이는지(ROI)를 정리했습니다.

📐
ROI 공식  =  ( 재작업·납품사고 비용 + 보고·취합 시간 + 유휴·과부하 인건비 + 문서 검색·입력 시간 ) 을 자동화·가시화로 줄인 금액  −  도입 비용

👑 관리자 (admin) — "시스템을 세팅해 조직 전체의 표준과 데이터를 통일한다"

구분내용
하는 작업계정·권한·기기 관리 / 사내 서버 연동(데이터 공유) / 프로세스·폴더 템플릿 표준화 / 브랜드·DB 설정
확인 포인트전체 대시보드 KPI(진행중·완료 프로젝트, 평균 건강도, 전체 작업 완료율) / 사용자 활성·기기 등록 상태
효용가치
(ROI)		 • 표준 템플릿 1회 정의 → 모든 프로젝트가 같은 체계로 시작 → 프로젝트마다 폴더·문서 체계를 새로 짜는 시간 제거
• 서버 공유로 데이터 단일화 → 파일 취합·버전 충돌·중복 입력 제거
• 전사 현황을 한 화면에 → 경영 보고 준비 시간 단축, 위험 프로젝트 조기 포착

🧭 매니저 (manager) — "프로젝트를 설계·운영해 지연과 누락을 막는다"

구분내용
하는 작업프로세스 단계 설계 / 작업 분배 + 담당자 선택 / 인력 배분(투입률) / 산출물·일정 관리 / 보고서 발행
확인 포인트건강도·진행률 / 분석의 일정 위험도(지연 GAP) / 누락 산출물 / 초과 배정 인원 / 단계별 진척
효용가치
(ROI)		 • 산출물 누락 자동 진단 → 납품 직전 사고·재작업 방지 (품질 손실 비용 ↓)
• 투입률·초과배정 경고 → 번아웃/유휴 인건비 회수, 즉시 인력 재배분
• 일정 위험도(지연 GAP) → 지연 조기 발견 → 납기 지연 패널티 ↓
• 보고서 자동 생성 → 주간보고·취합 회의 시간 ↓ (가장 직접적인 시간 ROI)

🙋 멤버 (member) — "내 작업에 집중하고 산출물을 자동으로 남긴다"

구분내용
하는 작업내 업무 확인 / 작업(칸반)에서 자기 카드 대기→진행중→검토중→완료 이동 / 산출물 파일 저장 후 ＋ 필수 등록 / 이슈·회의 기록
확인 포인트내 업무 목록·마감 임박(D-Day) / 내가 맡은 단계의 산출물 있음 / 누락
효용가치
(ROI)		 • 내 업무가 한 곳에 → "뭘 해야 하지" 찾고 묻는 시간 ↓
• 산출물 자동 발견 → 문서명 수기 입력 0 → 행정 잡무 ↓, 실무 시간 회수
• 진척이 자동 반영 → 따로 진척 보고를 안 해도 됨 → 보고 부담 ↓
✅
한 줄 요약: 관리자는 표준·데이터를 통일해 중복을 없애고, 매니저는 지연·누락을 미리 막아 사고 비용을 줄이며, 멤버는 잡무를 자동화해 실무에 집중합니다. 세 역할의 절감 시간·비용이 합쳐진 것이 곧 이 도구의 ROI입니다.

2 새 프로젝트 만들기

관리하려는 로컬 폴더를 지정하면, 그 폴더를 기준으로 진척·산출물을 분석합니다.

  1. 전체 대시보드 우측 상단의 ＋ 새 프로젝트 등록 클릭.
  2. 지역 코드 버튼을 눌러 광역시/도 → 시군구를 선택합니다. (필수)
  3. 프로젝트 유형 드롭다운에서 유형을 고릅니다. (선택)
  4. 프로젝트 명 에 이름을 입력합니다. (필수)
  5. 프로젝트 기간 달력에서 시작일~종료일을 선택합니다.
  6. 스캔할 로컬 폴더 경로 옆 폴더 선택 클릭 → 폴더를 고르면 경로가 자동 입력됩니다. (필수)
  7. 필요하면 적용할 폴더 양식(폴더·서류 자동 생성), 적용할 프로세스 템플릿(공정·일정 자동 설정)을 선택합니다. (선택)
  8. 프로젝트 생성 클릭. (지역코드·프로젝트명·폴더경로가 모두 있어야 활성화됩니다.)
💡
폴더 양식을 선택하면 표준 폴더 구조와 기본 문서 양식이 그 경로에 실제로 생성됩니다. 프로세스 템플릿을 선택하면 단계별 작업과 필수 산출물 체크리스트가 자동으로 만들어집니다.

2.5 실전 예제: "회사 홈페이지 구축" 프로젝트 ★

가상의 홈페이지 구축 프로젝트를 처음부터 끝까지 따라 하며, 폴더를 어떻게 만들고 어떻게 연결하는지를 구체적으로 설명합니다. (시나리오: "우리회사 홈페이지 리뉴얼", 2026-07-01 ~ 2026-09-30)

STEP 1. 먼저 로컬에 프로젝트 폴더를 만듭니다 (담당: 매니저/관리자)

이 앱은 실제 폴더를 스캔해서 진척과 산출물을 관리합니다. 그래서 먼저 윈도우 탐색기에서 폴더를 만들어 둡니다. 표준 웹 개발 단계(기획→디자인→개발→산출물)에 맞춰 아래처럼 구성하면, 뒤에서 "폴더 구조 적합도" 점수가 올라갑니다.

C:\Projects\Homepage_Renewal\      ← 이 폴더를 앱에 연결합니다
├── 01_기획\
│   ├── 요구사항정의서.docx
│   └── IA.xlsx               (메뉴 구조도)
├── 02_디자인\
│   ├── 디자인_시안.psd
│   └── logo.png
├── 03_개발\
│   ├── index.html
│   └── App.tsx
└── 04_산출물\
    ├── QA_보고서.pdf
    ├── 설명서.txt
    └── 버전관리.txt
ℹ️
처음엔 폴더만 만들고 파일은 비어 있어도 됩니다. 작업이 진행되면서 실제 파일을 채워 넣으면, 앱이 자동으로 "있음/누락"을 추적합니다. 폴더명 앞의 01_ 02_ 03_ 04_ 숫자는 단계 순서를 맞추는 데 도움이 됩니다.
⚡
폴더를 손으로 안 만들고 싶다면? 새 프로젝트 등록 시 적용할 폴더 양식을 선택하면 위 구조가 자동 생성됩니다. (폴더 양식은 매니저/관리자가 폴더 양식 라이브러리에서 미리 정의)

STEP 2. 앱에서 새 프로젝트 등록 + 폴더 연결 (담당: 매니저/관리자)

  1. 전체 대시보드 → ＋ 새 프로젝트 등록
  2. 지역 코드 선택 (예: 서울 → 강남구) 필수
  3. 프로젝트 유형 = W - 웹구축 선택 → 코드가 SG26W001 처럼 자동 생성됨
  4. 프로젝트 명 = 우리회사 홈페이지 리뉴얼 입력 필수
  5. 프로젝트 기간 = 2026-07-01 ~ 2026-09-30 선택
  6. 스캔할 로컬 폴더 경로 → 폴더 선택 → STEP 1에서 만든 C:\Projects\Homepage_Renewal 선택 필수 ← 이게 "폴더 연결"입니다
  7. 적용할 프로세스 템플릿 = 표준 웹 개발 템플릿 선택 → 기획/디자인/개발/산출물 단계와 필수 산출물 체크리스트(요구사항정의서.docx, 디자인_시안.psd 등)가 자동 생성됨
  8. 프로젝트 생성 클릭
🔗
"연결"이란? 6번에서 고른 폴더 경로가 프로젝트에 저장되고, 앱이 그 폴더를 스캔합니다. 즉 실제 폴더 ↔ 프로젝트 데이터가 묶입니다. 서버 공유 모드라면 이 경로는 모든 PC에서 같은 위치(예: 공유 드라이브 \\서버\Projects\Homepage_Renewal)여야 동료들도 같은 폴더를 봅니다.

STEP 3. 연결 확인 & 첫 스캔 (담당: 매니저/관리자)

  1. 생성된 프로젝트를 클릭 → 개요 탭으로 들어갑니다
  2. 빠른 작업의 폴더 구조 동기화 클릭 → 폴더를 다시 스캔
  3. 분석 탭에서 건강도 100점과 단계별 진척, 누락 항목을 확인

STEP 4. 산출물 채우기 — 자동 발견으로 등록 (담당: 멤버/매니저)

실제 작업 파일(요구사항정의서.docx 등)을 폴더에 넣은 뒤:

  1. 프로젝트의 산출물 화면으로 이동
  2. 상단 📁 폴더에서 자동 발견된 문서 카드에 폴더 속 파일이 자동으로 뜸
  3. 각 파일 ＋ 필수 등록 또는 ＋ 전체 등록 → 필수 산출물로 등록
  4. 이제 카드에 문서 완료율이 올라가고, 빠진 문서는 "누락"으로 표시됨
⚠️
자동 발견 목록이 비어 있으면 개요 → 폴더 구조 동기화로 다시 스캔하세요. (docx/xlsx/pdf/hwp/png 등 문서성 파일만 대상)

STEP 5. 단계·작업 분배와 진행 (역할별로 나뉨)

매니저/관리자가 하는 일① 프로세스 탭에서 단계(기획/디자인/개발/산출물) 확인·조정 → ② 작업 탭에서 작업 카드 추가 + 담당자 선택으로 멤버 지정 → ③ 프로젝트 인력 배분에서 투입 비율 설정
멤버가 하는 일① 내 업무에서 자기 작업 확인 → ② 작업 탭(칸반)에서 자기 카드를 다음 ▶으로 대기→진행중→검토중→완료 이동 → ③ 산출물 파일 폴더에 저장 후 STEP 4로 등록

작업이 완료될수록 단계 진행률과 프로젝트 건강도가 자동으로 다시 계산되고, 보고서 탭에서 진단서·성과 보고서를 뽑을 수 있습니다.

✅
한눈에 흐름: 폴더 만들기(매니저) → 프로젝트 등록·폴더 연결(매니저) → 프로세스 템플릿 적용(매니저) → 작업 분배(매니저) → 멤버가 작업 수행 + 산출물 채우기 → 자동 발견으로 등록(멤버) → 진척·건강도 자동 갱신 → 보고서(매니저).

4 프로세스(공정) 관리

프로젝트의 단계(기획·디자인·개발·산출물 등)를 정의합니다. (프로세스 탭, 관리자/매니저)

단계 추가

  1. 우측 상단 신규 단계 추가 클릭. (단계가 없으면 프로세스 첫 단추 생성)
  2. 단계 명칭 입력 (필수, 예: 05_퍼블리싱), 상세 설명 입력.
  3. 추가하기 클릭.

단계 편집 · 정렬

  • 카드의 편집(연필) — 진행 상태(대기/진행중/완료), 난이도(낮음~매우높음), 진행률, 프로세스 기간 설정 후 저장하기.
  • 위로 이동 / 아래로 이동 — 단계 순서 변경.
  • 단계 복제 / 단계 삭제.
ℹ️
단계 폴더명이 실제 폴더 구조와 일치하면 건강도의 폴더 구조 적합도 점수가 올라갑니다.

5 작업(칸반) 관리

단계별 세부 작업을 칸반 보드로 관리합니다. (작업 탭)

칸반 컬럼: 대기 진행중 검토중 완료

작업 추가하고 담당자 지정

  1. 상단에서 단계(프로세스) 탭을 먼저 선택합니다.
  2. 작업 카드 추가 클릭.
  3. 작업 제목 입력 (필수), 우선순위 선택(낮음/보통/높음/긴급).
  4. 담당자 선택 클릭 → 검색해 사용자 선택 → 확인.
  5. 작업 기간·시작 시간·종료 시간·설명 입력.
  6. 등록하기 클릭.

상태 옮기기

  • 카드의 다음 ▶ / ◀ 이전 버튼으로 컬럼을 이동합니다. (대기→진행중→검토중→완료)
  • 카드를 클릭하면 상세 패널이 열립니다 — 체크리스트 · 업무 이력 · 댓글 탭. 상태 변경 내역은 업무 이력에 자동 기록됩니다.
  • 상세 패널의 편집 으로 내용을 고치고 저장.
💡
작업을 완료로 옮기면 상위 단계의 진행률과 프로젝트 건강도가 자동으로 다시 계산됩니다.

폴더에서 자동 발견된 문서 등록하기

  2. 각 파일 옆 ＋ 필수 등록 을 누르면 해당 파일이 필수 산출물로 등록됩니다.
  3. 여러 개를 한 번에 올리려면 카드 우측 상단 ＋ 전체 등록 을 누릅니다.

등록된 문서 다루기

ℹ️
두 가지 문서 메뉴가 있습니다.
• 서류 양식 라이브러리 (전역) — 전사 공용 빈 서식을 올려두는 곳(수동 업로드).
• 프로젝트 산출물 (프로젝트 내부) — 그 프로젝트 폴더의 실제 문서를 자동 발견·추적하는 곳. 자동 발견은 여기서 동작합니다.
⚠️
자동 발견은 데스크톱 앱에서 실제 폴더가 연결돼 있을 때 동작합니다. 목록이 비어 있으면 개요 → 폴더 구조 동기화 로 다시 스캔하세요.

8 인력 배정

사람을 프로젝트에 투입하고 투입 비율(공수)을 관리합니다. (전역 배정 화면, 관리자/매니저)

인력 배정하기

  1. 신규 인력 배정 클릭.

  3. 투입 비율 을 슬라이더/숫자로 설정합니다. (0·25·50·75·100% 프리셋)

간편 편입(빠른 추가)

⚠️

11 설정 · 사용자 관리

브랜드, 서버 연동, 계정을 관리합니다. (환경 설정 / 사용자 계정 관리)

단독 사용 vs 데이터 공유 (중요)

기본은 각 PC가 독립적으로 데이터를 저장하는 단독 모드입니다. 여러 명이 같은 데이터를 공유하려면 한 대가 서버를 띄우고 나머지가 그 주소를 입력합니다.

  1. 환경 설정 → 사내 서버 연동 설정 으로 이동합니다.
  2. 사내 백엔드 서버 URL 에 서버 PC 주소를 입력합니다. (예: http://192.168.0.50:5000)
  3. 저장 및 반영 클릭 → 자동 새로고침됩니다.
ℹ️
서버에 연결되면 자동으로 서버 모드, 연결이 안 되면 자동으로 로컬 모드로 동작합니다. (수동 전환 버튼은 없습니다.) 현재 모드는 데이터 스토리지 항목에서 확인할 수 있습니다.

사용자 추가 (관리자)

  1. 사용자 계정 관리 → 우측 신규 계정 등록 카드.
  2. 사용자 이름·로그인 아이디·초기 비밀번호(6자+)·역할 입력. (부서/직급/직무 선택 가능)
  3. 계정 등록하기 클릭. 새 사용자는 첫 로그인 시 비밀번호를 바꿔야 합니다.

PC가 바뀐 사용자 — 장치 초기화

  1. 사용자 목록에서 해당 사용자의 상세/수정 클릭.
  2. 보안 영역에서 등록 PC 기기 강제 초기화 클릭 → 관리자 비밀번호 2차 인증.
  3. 이후 그 사용자가 새 PC에서 로그인하면 기기가 다시 등록됩니다.

12 자주 묻는 질문

Q. 동료에게 어떻게 주나요?

A. Folder Atlas_0.2.2_Portable.exe 파일 하나만 전달하면 됩니다. 받는 사람은 설치 없이 더블클릭만 하면 됩니다.

Q. 내 데이터는 어디에 저장되나요?

A. 단독 모드에서는 내 PC의 AppData/Roaming/com.folder-atlas.app/ 안의 데이터베이스 파일에 저장됩니다. 다른 PC와 공유되지 않습니다.

Q. 산출물 자동 발견 목록이 비어 있어요.

A. 프로젝트 폴더 경로가 실제 폴더를 가리키는지 확인하고, 개요 탭의 폴더 구조 동기화 를 눌러 다시 스캔하세요. 문서성 파일(docx/xlsx/pdf/hwp/png 등)만 발견 대상이며 node_modules 등 시스템 폴더는 제외됩니다.

Q. 비밀번호를 잊었어요.

A. 관리자(admin)가 사용자 계정 관리 → 상세/수정 → 임시 비밀번호 설정 으로 초기화해 줄 수 있습니다.

Q. 실행 시 보안 경고가 떠요.

A. 사내 미서명 앱이라 정상입니다. 추가 정보 → 실행 을 누르세요.
